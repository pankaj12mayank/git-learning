# 🧠 1️⃣ What Is CI/CD?

### 🔹 CI (Continuous Integration)

Every time code is pushed:

* It builds
* It runs tests
* It validates quality

Automatically.

### 🔹 CD (Continuous Delivery / Deployment)

After successful build:

* Code is prepared for release
* Or deployed automatically

---

# 🏗 What Is a Pipeline?

Pipeline = Automated steps executed in sequence.

Example:

```
Step 1: Checkout Code
Step 2: Install Dependencies
Step 3: Run Tests
Step 4: Build Project
Step 5: Deploy
```

---

# ⚙️ 2️⃣ What Is GitHub Actions?

GitHub Actions = Built-in CI/CD engine inside GitHub.

It runs workflows defined in:

```
.github/workflows/
```

Using YAML file.

---

# 🧪 Let’s Create First Pipeline (Hands-On)

## Step 1 – Create Workflow Folder

In your local project:

Create folder:

```
.github/workflows
```

Inside that create file:

```
ci.yml
```

Path should be:

```
.github/workflows/ci.yml
```

---

## Step 2 – Add This Basic Workflow Code

Paste this inside `ci.yml`:

```yaml
name: Basic CI Pipeline

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout Code
        uses: actions/checkout@v3

      - name: Run a simple command
        run: echo "Pipeline is running successfully!"
```

Save file.

---

## Step 3 – Commit & Push

```
git add .
git commit -m "Added basic CI pipeline"
git push
```

---

## Step 4 – See Pipeline Running

Go to GitHub repo → Click **Actions** tab.

You’ll see pipeline running.

If green ✅ → success
If red ❌ → failed

# 🔐 Connect with Branch Protection

Go to:

Settings → Branches → Edit protection rule

Enable:

✔ Require status checks to pass before merging

Select your pipeline.

Now PR cannot merge unless CI passes.

That’s enterprise-level governance.
